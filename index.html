BC VenWolf Adapter

Install the userscript that bridges Bondage Club activity events to VenWolf broadcast control.

Userscript Installation

Install Tampermonkey, Violentmonkey, or another userscript manager, then use the production script below.

Install production version View raw script GitHub repository

Production version: https://github.com/QAQMOON/-BC-VenWolf-Adapter/raw/main/bc-venwolf-adapter.user.js

Automatic Updates

The userscript metadata includes stable update URLs:

  • @updateURL points to the raw GitHub script.
  • @downloadURL points to the raw GitHub script.

For each release, update both @version and the internal VERSION constant before pushing to GitHub. Userscript managers use the version number to decide whether to update.

// @updateURL    https://raw.githubusercontent.com/QAQMOON/-BC-VenWolf-Adapter/main/bc-venwolf-adapter.user.js
// @downloadURL  https://raw.githubusercontent.com/QAQMOON/-BC-VenWolf-Adapter/main/bc-venwolf-adapter.user.js

Quick Start

VenWolf config should allow broadcast control:

host: "0.0.0.0"
port: 8920
allowBroadcastToClients: true

After entering a Bondage Club room, run:

/vw status
/vw dry on
/vw test 20 3000
/vw dry off

Version 0.1.2 registers commands through BC Mod SDK when available. After entering a room you should see [VenWolf] adapter ready v0.1.2; then /vw status should be handled locally.

Enable GitHub Pages

  1. Open the repository settings.
  2. Go to Pages.
  3. Choose Deploy from a branch.
  4. Use branch main and folder /root.

After GitHub Pages is enabled, this installation page will be available at https://qaqmoon.github.io/-BC-VenWolf-Adapter/.
